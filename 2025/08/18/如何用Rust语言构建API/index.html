用 Rust 和 Actix-web 从零开始写一个 RESTful API

目标读者：有 Rust 基础、第一次用 Actix-web 的同学
阅读时间：20 分钟
最终成果：一个能跑在本地 8080 端口、带 CRUD 的 Todo API，并附带完整的目录结构与测试用例。

1. 为什么要选 Actix-web？

Actix-web 是目前 Rust 生态里性能最好、文档最完整、第三方支持最多的 Web 框架之一。
它基于 actor 模型，配合 async/await，QPS 实测能轻松破 100k（当然，我们的重点还是“开发体验”）。

2. 环境准备

cargo 随 rustc 自带 包管理
PostgreSQL 任意 数据持久化（本文使用 SQLx + Postgres）
cargo-watch 可选 热重载

3. 运行命令

  • cargo run 编译 + 执行
  • 第一次会下载依赖并编译；后续增量编译只要几秒。

4. 测试请求

  • -X POST 指定 HTTP 方法
  • -H 设置请求头，告诉服务器发送的是 JSON
  • -d 指定请求体
  • 服务器收到后原样返回，验证链路正常。

总结一句话

每行代码都是最小必要步骤：

  • 框架依赖 → 路由宏 → 处理器 → 服务器启动 → 运行测试。
